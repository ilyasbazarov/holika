## Epic M — Миграция на docs-driven метод *(текущий)*

Цель: перевести проект с `_SOURCE_MONOLITH.md` на нумерованный набор доков + процесс ролей, фазами,
без big-bang, с полной трассируемостью. Полный план и приёмка фаз — в `TZ-MIGRATE-TO-DOCS-METHOD.md`.

| Задача | Цель | Приёмка (кратко) |
|---|---|---|
| M-P0 | Каркас ролей + заморозка монолита | каркас в репо; монолит заморожен; 0 фактов проекта в каркасе |
| M-P1 | `_METHOD.md` | доменно-нейтрален; покрывает инварианты метода — **DONE** |
| M-P2 | Инвентаризация + `MIGRATION_MAP.md` | каждая единица монолита имеет вердикт; покрытие 100%; GAP-реестр в `07` |
| M-P3 | Скелеты `00`–`03`,`09` (`00_CHARTER` первым) | карта документов ↔ скелеты; ERP-граница в `00`/`02`; первый discovery-бриф на выгрузку эталона |
| M-P4 | Наполнение доков | трассируемость к монолиту; 0 фактов без источника; противоречия → ADR-кандидаты |
| M-P5 | Процессный слой | ретро-ADR в `06`; `07` полон; первый боевой бриф; discovery-очередь приоритизирована |
| M-P6 | Cutover + e2e процесса | матрица 100%; источник истины → доки; реальная задача прошла полный цикл ролей |

**Зависимости:** M-P2 ← коммит `_SOURCE_MONOLITH.md` (блокер B-1 в `07`). M-P4 ← M-P3 ← M-P2.
