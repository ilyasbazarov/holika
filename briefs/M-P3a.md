`00_CHARTER` полностью, т.к. уже зафиксированы в ADR.

## Шаги
1. **Скоуп — ровно 5 файлов:** `00_CHARTER.md`, `01_ARCHITECTURE.md`, `02_ERP_CONTRACTS.md`,
   `03_PIPELINE_SPEC.md`, `09_GLOSSARY.md`. Скелеты `10_OPS_PLAYBOOK` и `11_INFRA_FACTS` — НЕ в этой задаче
   (вне scope, см. ниже), но в карте документов `00_CHARTER` они перечисляются.
2. Для каждого из 5 доков: из `MIGRATION_MAP.md` собери множество секций = единицы, чей вердикт указывает
   на этот док (напр. в `02_ERP_CONTRACTS` идут PR-16/23/24/25/26/27/28/32/33ч/34ч/35ч, RB-18ч; §-названия —
   по колонке вердикта: «§схемы stg», «§схемы core», «§схемы dim», «§поведение МойСклад API», «§справочные
   данные», «§оракул» и т.д.). Каждая секция = заголовок + одна строка `<!-- P4: из PR-xx / RB-yy -->`.
   Прозу единиц НЕ переносить.
3. **Решённые якоря `00_CHARTER` — писать ПОЛНОСТЬЮ (не stub):**
   - §миссия/контекст — из PR-01 (BI для инвестора; дистрибьютор корейской косметики, Бишкек; рынки
     Дордой/Джунхай, ~99% B2B).
   - §главный принцип = **граница контракта** — ERP-референс-оракул (**ADR-002**): выход обязан биться с
     эталонными цифрами МойСклада «по цифрам»; расхождение закрывается правилом/override/ADR, не тихим
     примирением. Эхо словами владельца — PR-34 правило 34 («ground truth только из МойСклада»).
   - §карта документов — полная таблица `00`–`11` + `/reference` с изменчивостью (STABLE/SEMI-STABLE/LIVING/
     append), **строго по таблице «Целевой набор доков» из `MIGRATION_MAP.md`**, включая `11_INFRA_FACTS`
     (ADR-004 accepted).
   - §scope in/out и §критерии успеха — каркас с пунктами, выводимыми из миссии и границы; факты, которых нет
     в источнике/карте, НЕ выдумывать → `CONTEXT GAP` на конкретный пункт.
4. **`02_ERP_CONTRACTS` §оракул** — создать секцию-stub с явным местом под регистрацию `/reference`-артефактов
   (её наполняет задача **M-P3b**). Указатель: `<!-- P3b: регистрация /reference (seed pnl_2026-05) -->`.
5. Проверь инвариант приёмки: каждый док из карты имеет либо скелет-файл (эти 5), либо строку в карте
   документов `00_CHARTER` (все 12). Каждая секция скелета трассируется к ≥1 единице карты. Нет «сиротских»
   секций без единицы и нет единиц (с вердиктом «→ этот док»), не попавших ни в одну секцию.
6. Имена/идентификаторы — EN snake_case; проза RU (инвариант `05` Часть II). Точные имена доков — из карты,
   не изобретать (`03` = именно `03_PIPELINE_SPEC`).
