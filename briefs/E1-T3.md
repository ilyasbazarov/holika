3. **Слой B — `core.fact_payments` (read-only bq query).** Тот же скоуп статьи, `moment` ∈ май. Верни: Σ `sum_kgs`, COUNT, построчно `payment_id`/`moment`/`sum_kgs`/`_loaded_at`, и **отдельно** COUNT строк этой статьи с `moment IS NULL` (прод-SQL их роняет через `WHERE p.moment IS NOT NULL`).
   ```bash
   bq query --use_legacy_sql=false --project_id=msklad-bi-prod --format=prettyjson '
   SELECT
     COUNT(*) AS n_rows,
     COUNTIF(moment IS NULL) AS n_moment_null,
     ROUND(SUM(sum_kgs), 2) AS sum_kgs_total
   FROM `msklad-bi-prod.core.fact_payments`
   WHERE expense_item_name = <verbatim-имя из expense_articles_client.md>
     AND DATE(moment) BETWEEN "2026-05-01" AND "2026-05-31"'
   ```
   (Замени `<verbatim-имя…>` строковым литералом из `expense_articles_client.md` — это НЕ секрет-плейсхолдер, а значение из репо; регистр критичен.)

4. **Слой C — `marts.expenses` (read-only), воспроизведи прод-цифру.** Повтори ровно тот фильтр мая, которым E1-T2 §4 получил **3 355 976,16**, и зафиксируй КАКОЙ предикат месяца применялся (`year_month='2026-05'` / `month_start` / `DATE(moment)`), т.к. именно тут живёт TZ-кандидат.
   ```bash
   bq query --use_legacy_sql=false --project_id=msklad-bi-prod --format=prettyjson '

   FROM `msklad-bi-prod.marts.expenses`

   ```

5. **Декомпозиция разрыва по слоям.** Собери таблицу A/B/C (Σ + doc-set) и разложи 890 187,50:
   - **Δ(A↔B)** — расхождение оракул↔core → апстрим: недозагруженные/удалённые документы. Выпиши `id`, отсутствующие/лишние в B против A. Для каждого недостающего проверь `_loaded_at` в core.
   - **Δ(B↔C)** — расхождение core↔март → даунстрим: месячный бакетинг. Для каждого документа посчитай месяц под **UTC** vs под **Asia/Bishkek (UTC+6)** — выпиши те, у кого бакет «перескакивает» границу мая (`moment` в пределах ±6 ч от края месяца), и `moment IS NULL`-строки, роняемые прод-SQL.
   - Инвариант: `Δ(A↔B) + Δ(B↔C) = 890 187,50` без необъяснённого остатка. Остаток есть → не закрывай, копай (или `CONTEXT GAP`).
