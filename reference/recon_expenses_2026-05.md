## ⚠️ Вне 26 категорий — «Вывод прибыли» (капитал, ADR-013 §1) — АНОМАЛИЯ, требует адъюдикации

|---|---:|---:|---:|
| Вывод прибыли | 4 246 163,66 | 4 246 163,66 | **0,00** |

Прод и оракул сходятся **точно**. Это противоречит `ADR-016` (§Контекст/§2): `E1-T3` доказал по прямому diff `core.fact_payments` vs **live API** для этой же статьи/периода — 38/38 не-KGS `paymentout`-документов мая занижены (Δ(A↔B)=890 187,50), системное занижение. Три независимых цифры не бьются друг с другом:
- (A) BQ `core.fact_payments` (= прод `marts.expenses`, эта сверка) — 4 246 163,66
- (B) live МойСклад API (E1-T3, `recon_vyvod_pribyli_2026-05.md`) — доказанно занижена на 890 187,50 относительно
- (C) оракул — ручная выгрузка владельца из UI, 2026-07-06 — 4 246 163,66, **совпадает с (A), не с ожидаемой (A+890 187,50)**

Не примиряю тихо. Гипотезы (не выбираю — фактов недостаточно): (i) оракул (C) сам по себе построен на другом определении/срезе документов, чем API-diff (B), и потому не является независимым подтверждением бага для ЭТОЙ конкретной сверки; (ii) `recon_vyvod_pribyli_2026-05.md` (не в контексте этой сессии) может содержать разбивку, снимающую видимое противоречие. Оба варианта требуют чтения `recon_vyvod_pribyli_2026-05.md`, которого нет в context-to-load `E1-T1` → `CONTEXT GAP`, зарегистрирован как `Q-46` (см. ## Гипотеза ADR-012 §4 — вердикт

**ПОДТВЕРЖДЕНА числом, частично.** Структурное занижение из-за отсутствия `loss` и `commissionreportin` — доказано на 3 из 4 ненулевых категориях точным (до копейки, без остатка) совпадением: «Списания» (100% на loss), «Расходы маркетплейсов» (Δ = ровно сумма «Комиссии»), «Прочие расходы» (согласуется с 1 loss-документом по Q-9). Для «Маркетинг и реклама» гипотеза MISSING_SOURCE — доминирующая, но не исключён остаточный `FX_BUG`-компонент (не измерен в этой сессии). Ни один из 4 гэпов не отнесён к `FX_BUG` с уверенностью — по имеющимся данным майская сверка 26-категорий закрывается целиком через `MISSING_SOURCE`, без обнаруженного остатка `FX_BUG` внутри этого среза (это НЕ опровергает ADR-016 — систематика бага установлена отдельным методом, live-API diff, не этой оракул-сверкой; см. аномалию по «Вывод прибыли» выше).
