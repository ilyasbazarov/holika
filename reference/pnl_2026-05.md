## Правило сверки (из ADR-005, `proposed`)

1. **Гранулярность — статья П&Л (line-level), не только итог.** Разрыв бьёт именно на уровне статей (4 отсутствующие статьи = 85,6% разрыва); итог-only потерял бы диагностику.
2. **Точное совпадение (до копейки) на величинах класса `ORACLE`.** Величины `PIPELINE_SNAPSHOT` не сверяются — это измеряемая сторона, а не эталон.
3. **Целевая сверка расходов/платежей:** `fact_payments` (+ будущая загрузка `entity/loss`) → **11 132 303,89**.
4. **Известный разрыв НЕ примиряется тихо.** Строка `fact_payments` ↔ `11 132 303,89` привязана к **зарегистрированному открытому исключению**:
   - **Q-8** — методология агрегации П&Л МойСклада (остаток `523 654,81`, размазан; TD-PNL-RECON-01).
   - **Q-9 / Q-10** — 4 статьи (`3 115 805,82`) физически живут в `entity/loss`, который `cf-finance` не запрашивает вовсе.
   - **Допуск** для этой строки = документированный разрыв, **пока Q-8 не закрыт**. Закрытие исключения = discovery Q-8 → новый ADR, не подгонка.
5. **Любое расхождение, не покрытое зарегистрированным исключением, — стоп** + правило / override / ADR (`_METHOD.md` §4b), никогда не тихая подгонка «более правдоподобной» половины.

## Сохранённые оговорки источника (не сглаживать)

- **«Неразнесенное списание»** на 2026-06-25 показывает **0 строк за май** даже на свежей перезагрузке `cf-finance` — при том, что П&Л МойСклада даёт по этой статье `1 260 262,00`. Гипотеза «устаревшие данные» опровергнута (TD-PAYMENTS-RECOUNT).
- Часть статей (`Списания`, `Комиссия`, `Обучение`, `Неразнесенное списание`) физически живёт в документах списания (`entity/loss`), которые `cf-finance` **не запрашивает в принципе** (TD-PNL-RECON-01).
- Прямая арифметика `entity/loss` тоже **не сходится 1:1** с П&Л МойСклада — методология агрегации не установлена (**связь с Q-8**, отдельный discovery-бриф).
