## §главный принцип — граница контракта (референс-оракул ERP)

**Красная линия проекта — референс-оракул приёмки** (`_METHOD.md` §4b), зафиксированный **ADR-002**:

> Выход проекта (`core.*` / `marts.*` / витрины Looker Studio) **обязан биться с эталонными цифрами МойСклада «по цифрам»**. Любое расхождение закрывается **правилом, override-ом или ADR — никогда тихим примирением** «более правдоподобной» половины.

**Источник истины реконсиляции — ТОЛЬКО прямой запрос к API МойСклада**, никогда собственные BQ-таблицы (**слова владельца, PR-34 правило 34**). Сравнение `core.*`/`marts.*` между собой проверяет лишь внутреннюю согласованность пайплайна — это **не** проверка полноты данных. Чтобы найти, что не доезжает от МойСклада, нужен независимый прямой запрос к `api.moysklad.ru` за тот же скоуп, и только затем diff с BQ.

**Закон-следствие:** слой не лезет через границу чинить чужой слой — дефект правится **фикс-форвардом в нужный слой**.

Точный состав эталонных цифр (ground truth) выгружается руками в `/reference` (первый seed — П&Л МойСклада май-2026, PR-28) — **discovery Q-1, задача M-P3b**. До закрытия Q-1 приёмка проверяет трассируемость к источнику; сходимость к ERP «по цифрам» включается после посева эталона.

## §scope — in / out

**IN scope** (выводимо из миссии и границы):
- BI-пайплайн выгрузки из **МойСклад ERP → GCS/BQ (слои `stg` / `core` / `marts`) → Looker Studio**. <!-- P4: из RB-03 (топология), 01_ARCHITECTURE -->
- **Реконсиляция выхода с эталонными цифрами МойСклада** «по цифрам» (граница контракта, ADR-002). <!-- P4: из ADR-002, PR-28, PR-34 правило 34 -->
- Контроль качества данных (DQ-gate) и марты для инвесторской оценки. <!-- P4: из PR-19/PR-31 (DQ), PR-29/PR-30 (marts) → 03_PIPELINE_SPEC -->

**OUT of scope / исключения:**
- Данные-исключения на уровне пайплайна: **«Перемещение исходящий»** (легитимно исключено из ETL, PR-28) и **`EXCLUDE_EXPENSE_IDS`** (системные статьи расходов). <!-- P4: из PR-28, PR-27/PR-32; полный список EXCLUDE_EXPENSE_IDS — GAP Q-10 -->
- `CONTEXT GAP`: **проектный scope-out** (что BI-система сознательно НЕ делает) в источнике/карте явно не перечислен — не выдумывать; уточнить в P4/discovery либо оставить `DEFER` в `07_STATE`.

## §критерии успеха

- **Сходимость с оракулом:** выход бьётся с эталонными цифрами МойСклада «по цифрам»; каждое расхождение объяснено и закрыто правилом/override/ADR (ADR-002). <!-- P4: из ADR-002; включается после закрытия Q-1 (M-P3b) -->
- **Трассируемость:** каждый факт целевого дока трассируется к единице `MIGRATION_MAP` → строке источника (инвариант метода). <!-- P4: инвариант _METHOD §6 -->
- **DQ-приёмка:** прохождение DQ-gate по зафиксированным порогам. <!-- P4: из PR-19/PR-31/PR-34 правило 25 → 03_PIPELINE_SPEC §DQ -->
- `CONTEXT GAP`: **инвесторские метрики успеха** (конкретные пороги «готовности к инвестору») в источнике не заданы — не выдумывать; уточнить с владельцем в P4/discovery.
