## §семантика П&Л

**ADR-006 (M-P3c, 2026-07-06): Методология агрегации расходной части П&Л МойСклада**

Расходная строка П&Л МойСклада строится из двух источников:

**Источник #1: paymentout + cashout + loss (22 статей)**
- Три вида документов МойСклада: `paymentout` (исходящие платежи), `cashout` (выплаты наличностью), `entity/loss` (прямые потери/написания).
- Валютная конвертация: каждый документ конвертируется по курсу валюты самого документа (rate.value из масива `rate`), не по текущему курсу.
- Примеры статей: Расходы маркетплейсов, Канцелярия и оргтехника, Зарплата, Содержание авто, Реклама и маркетинг, Топливо и смазки, Благоустройство.
- Методология подтверждена точным совпадением (0.00 разницы) всех 22 статей с ручной выгрузкой П&Л 2026-07-06 (эталон ADR-005, seed #2).

**Источник #2: entity/commissionreportin (1 статья — Комиссия)**
- Статья строится из таблицы `entity/commissionreportin` (специальная сущность для комиссионных вознаграждений).
- Формула: Σ(reward÷100) + Σ(commissionOverhead.sum÷100) по всем документам периода.
- Примечание: поля `reward` и `commissionOverhead` хранят суммы в **базовых единицах** (как и все суммы в МойСклада), отсюда ÷100 для конвертации в валюту. 
- Методология подтверждена на 7 документах мая-2026 с точным совпадением (0.00 разницы).

**На дашборде (27 категорий):**
- Категория «Расходы маркетплейсов» объединяет **обе** статьи: источник #1 (одноимённая статья) + источник #2 (entity/commissionreportin).
- Остальные 6 статей (Перемещение исходящий, Благотворительность, Возврат займа собственнику, Выплата тела кредита, Проценты по кредиту, Возврат) исключаются из дашборда как движения капитала/финансирования (не операционные расходы).
- Нулевые значения по статьям (IT, Аренда, Фулфилмент в мае-2026) — ожидаемы (подтверждено эталоном).

**Закрытие Q-8:** ORACLE seed #1 (ADR-005, 2026-06-25) заменяется на seed #2 (свежий эталон 2026-07-06 из МойСклада), поскольку seed #1 содержал ошибки ввода (гипотеза дрейфа отклонена).
