## §семантика П&Л

**ADR-006 (M-P3c, 2026-07-06): Методология агрегации расходной части П&Л МойСклада**

Расходная строка П&Л МойСклада строится из двух источников:

**Источник #1: paymentout + cashout + loss (30 из 31 статьи)**
- Три вида документов МойСклада: `paymentout` (исходящие платежи), `cashout` (выплаты наличностью), `entity/loss` (прямые потери/списания).
- Валютная конвертация: каждый документ конвертируется по курсу валюты самого документа (`rate.value` из объекта `rate`), не по текущему курсу.
- Подтверждённые статьи (полный список, точное совпадение 0.00 разницы с эталоном 2026-07-06): Списания, IT, Аренда, Банк (комиссия), Бонусы, Бухгалтерские услуги, Зарплата, Интернет и связь, Коммунальные услуги, Логистика, Маркетинг и реклама, Мой Склад, Неразнесенное списание, Нотариальные и юридические услуги, Обучение, Офисные расходы, Охрана, Покупка основных средств, Прочие расходы, Расходы маркетплейсов, Ремонт оборудования, Таможенные расходы, Техника, Топливо, Фулфилмент (плюс статьи вне дашборд-scope: Благотворительность, Возврат займа собственнику, Вывод прибыли, Выплата тела кредита, Проценты по кредиту, Перемещение исходящий).
- Методология подтверждена точным совпадением (0.00 разницы) всех статей этой группы с ручной выгрузкой П&Л 2026-07-06 (эталон ADR-006, seed #2).

**Источник #2: entity/commissionreportin (1 статья — Комиссия)**
- Статья строится из таблицы `entity/commissionreportin` (специальная сущность для комиссионных вознаграждений от маркетплейсов).
- Формула: Σ(reward÷100) + Σ(commissionOverhead.sum÷100) по всем документам периода.
- Примечание: поля `reward` и `commissionOverhead` хранят суммы в **минорных единицах** (как и все денежные поля МойСклад), отсюда ÷100.
- Методология подтверждена на 7 документах мая-2026 с точным совпадением (0.00 разницы).

**На дашборде (26 категорий, список заказчика):**
- Категория «Расходы маркетплейсов» объединяет **обе** статьи: источник #1 (одноимённая статья) + источник #2 (`entity/commissionreportin`).
- 6 статей исключаются из дашборда как движения капитала/финансирования (не операционные расходы): Перемещение исходящий, Благотворительность, Возврат займа собственнику, Выплата тела кредита, Проценты по кредиту, Вывод прибыли.
- Нулевые значения по статьям (IT, Аренда, Неразнесенное списание, Обучение, Таможенные расходы, Фулфилмент в мае-2026) — ожидаемы (подтверждено эталоном 2026-07-06), не пробел в данных.

**Закрытие Q-8:** ORACLE seed #1 (ADR-005, 2026-06-25) заменён на seed #2 (свежий эталон 2026-07-06 из МойСклада, `/reference/pnl_2026-05.md`), поскольку seed #1 содержал ошибки ручного ввода (гипотеза дрейфа данных отклонена владельцем).
