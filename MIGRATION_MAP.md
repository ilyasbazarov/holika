# MIGRATION_MAP — карта переноса источника на docs-driven метод

**Фаза:** M-P2 (ТЗ §5) · **Дата:** 2026-07-03 · **Автор:** архитектор
**Источники (заморожены):** `PROJECT_REFERENCE_v6.md` (далее **PR**, 867 строк) + `RUNBOOK_v8.md` (далее **RB**, 1296 строк) @ SHA `8a17b84be1d49387d8387a8503019dba3a8f8eb6`
**Правило:** каждая единица источника имеет ровно один вердикт. Проза НЕ переписывается в этой фазе — только адресация.

## Легенда вердиктов

- **→ NN_DOC §…** — переносится в целевой док (наполнение — в P4, с трассировкой к строкам ниже).
- **DROP (причина)** — не переносится; остаётся читаемым в замороженном источнике-архиве.
- **DROP-DUP → ед. X** — дубликат другой единицы; переносится один раз, по вердикту единицы X.
- **DEFER (куда/когда)** — сознательно откладывается, с точкой возврата.
- **GAP Q-n (тип)** — знания в источнике не хватает; регистрируется в `07_STATE` «Открытые вопросы»; тип: факт / решение / референс-артефакт.

## Целевой набор доков (имена финализируются в P3 через `00_CHARTER`)

| Док | Роль в этом проекте | Изменчивость |
